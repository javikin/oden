Metodología Documentation-First

Un sistema completo para crear proyectos profesionales documentando y diseñando COMPLETAMENTE antes de escribir código.

Oden Forge es un toolkit para Claude Code que implementa la metodología Documentation-First Development, probada en múltiples proyectos reales con resultados medibles.

¿Qué es Documentation-First?

Una metodología de desarrollo que invierte la relación tradicional entre código y documentación. En lugar de documentar después de codificar, documentamos antes.

Principios Core

  1. Documentar TODO antes de codificar - Arquitectura técnica, análisis competitivo, especificaciones detalladas
  2. 8,000+ líneas de documentación antes de la primera línea de código
  3. Design Sprint Adaptado - Diseño rápido → Validación → Iteración
  4. Entrega incremental - Valor tangible cada semana

Fases del Proceso

FASE 1: Pre-Desarrollo (1-2 semanas)

  • Technical Architect: technical-decisions.md (2000-4000 líneas), schema de BD, interfaces TypeScript
  • Domain Expert: Análisis de 3-5 competidores, user personas, priorización de features
  • Spec Writer: Especificaciones de 800-1200 líneas por módulo, máquinas de estado, edge cases
  • Implementation Planner: Plan semana por semana con estimaciones y milestones

FASE 2: Implementación (8-18 semanas)

  • Seguir especificaciones al pie de la letra
  • Daily logging de progreso
  • Validación continua contra specs

FASE 3: Post-Desarrollo

  • Documentar lecciones aprendidas
  • Crear guías de usuario
  • Archivar documentación obsoleta

Comandos Disponibles

Oden Forge proporciona comandos especializados para cada fase del desarrollo:

Setup y Configuración

/oden:init      # Wizard interactivo completo
/oden:mcp       # Gestionar MCPs (install, status, update)
/oden:help      # Documentación y guías

Pre-Desarrollo (ejecutar en orden)

/oden:architect # Technical decisions + DB schema (2000+ líneas)
/oden:prd       # Product requirements con análisis competitivo
/oden:epic      # PRD → Epic técnico con work streams

Desarrollo

/oden:work      # Orquestador con Teams paralelos
/oden:debug     # Queue-based debug orchestration
/oden:sync      # Sincronización con GitHub Issues

Utilidades

/oden:research  # Investigación técnica
/oden:daily     # Registrar progreso diario
/oden:review    # Code review antes de merge

Instalación y Uso

1. Instalar

npm install -g oden-forge

2. Inicializar en tu proyecto

cd tu-proyecto
/oden:init

3. Seguir la metodología

# Pre-desarrollo
/oden:architect
/oden:prd feature-name
/oden:epic feature-name

# Desarrollo
/oden:work feature-name
/oden:debug

Por Qué Funciona

Métricas Comprobadas

  • 8,000+ líneas de documentación antes del código
  • 90%+ success rate en debugging estructurado
  • 8-20 semanas de MVP a producto enterprise
  • 95%+ context preservation entre sesiones

Beneficios

  • Reduce deuda técnica: Las decisiones están documentadas desde el inicio
  • Acelera desarrollo: Los desarrolladores saben exactamente qué construir
  • Mejora comunicación: Stakeholders ven el progreso en documentación clara
  • Facilita mantenimiento: La arquitectura está completamente documentada
