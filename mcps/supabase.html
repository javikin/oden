MCPs / Supabase

🗄 Supabase

Backend Type: NPX Access Token required

One-Click Install

Claude Code 
/oden:init-mcp backend

Manual Configuration

You'll need your Supabase Access Token and Project Reference ID.

Get your Access Token

Go to your Supabase Dashboard > Account > Access Tokens. Create a new token and copy it.

Get your Project Reference

Go to your project Settings > General. Copy the Reference ID (looks like abcdefghijklmnop).

Add to .mcp.json

json - .mcp.json 
{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": [
        "-y",
        "@supabase/mcp-server-supabase@latest",
        "--read-only",
        "--project-ref=YOUR_PROJECT_REF"
      ],
      "env": {
        "SUPABASE_ACCESS_TOKEN": "YOUR_ACCESS_TOKEN"
      }
    }
  }
}
⚠️
The --read-only flag is recommended for safety. Remove it only if you need Claude to execute write queries.

What It Does

  • Query your database schema (tables, columns, relationships)
  • Run read-only SQL queries to inspect data
  • View RLS policies and triggers
  • Understand your data model without switching contexts
  • Generate TypeScript types from your schema

Usage Examples

Prompt examples 
# Inspect schema
"Show me all tables and their relationships in my Supabase project"

# Check data
"How many active users do I have in the last 30 days?"

# Generate types
"Generate TypeScript interfaces matching my orders table schema"

# Review policies
"List all RLS policies on the orders table"

Best Practices

  • Always use --read-only unless you specifically need write access
  • Store your access token securely - don't commit .mcp.json with real tokens
  • Add .mcp.json to .gitignore if it contains secrets
  • Use a dedicated access token with minimal permissions
