Installation Guide

Step-by-step setup for Oden Forge MCPs

Prerequisites

  • Claude Code installed and working
  • Git installed (git --version)
  • Node.js (for NPX-based MCPs) - v18+ recommended
  • GitHub CLI (optional, for sync features) - gh auth status

Method 1: One-Click with Oden Forge

The fastest way. Install Oden Forge and use the built-in MCP installer.

Clone and install Oden Forge

bash 
git clone https://github.com/javikin/oden.git my-project
cd my-project
./install.sh

Open Claude Code and install MCPs

Claude Code 
# Install essential MCPs
/oden:init-mcp essential

# Or install by category
/oden:init-mcp design
/oden:init-mcp backend
/oden:init-mcp testing
/oden:init-mcp mobile
/oden:init-mcp devops
/oden:init-mcp productivity

Restart Claude Code

MCPs load on startup. Exit Claude Code and reopen it. Verify with /mcp.

Method 2: Manual Configuration

For projects that don't use Oden Forge, or if you prefer manual control.

Create .mcp.json in your project root

bash 
touch .mcp.json

Add MCP configurations

Choose a stack template or pick individual MCPs from the catalog.

json - .mcp.json 
{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    }
  }
}

Add more MCPs as needed

Each MCP is an entry inside the mcpServers object. See individual MCP pages for the exact configuration.

Restart Claude Code and verify

Run /mcp inside Claude Code to see connected servers.

Global vs Project Configuration

MCPs can be configured at two levels:

Project Level (recommended)

File: .mcp.json in project root
Only loads for this specific project. Best for project-specific MCPs like Supabase.

Global Level

File: ~/.claude.json
Loads for all projects. Best for universal MCPs like Context7.

⚠️
If your .mcp.json contains sensitive data (API keys, tokens), add it to .gitignore to avoid committing secrets.

MCP Types

HTTP MCPs

Connect to remote services via URL. Authentication is usually handled via OAuth in the browser.

json 
{
  "name": {
    "type": "http",
    "url": "https://mcp.service.com/mcp"
  }
}

NPX MCPs

Run locally via npx. May require environment variables for authentication.

json 
{
  "name": {
    "command": "npx",
    "args": ["-y", "@scope/mcp-server@latest"],
    "env": {
      "API_KEY": "your-key"
    }
  }
}
