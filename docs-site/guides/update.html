Update Guide

Keep your MCPs and Oden Forge up to date

Updating MCPs

NPX-based MCPs (auto-update)

MCPs that use npx -y @scope/package@latest automatically fetch the latest version each time they start. No action needed.

✅
MCPs configured with @latest in their args are always up to date. This includes: Supabase, Playwright, Chrome DevTools, iOS Simulator.

HTTP-based MCPs

HTTP MCPs (Context7, Figma, Vercel, Notion) connect to hosted services. They are updated server-side automatically. No action needed.

Pinning specific versions

If you need a specific version for stability:

json 
// Instead of @latest (auto-updates)
"@supabase/mcp-server-supabase@latest"

// Pin to specific version (manual updates needed)
"@supabase/mcp-server-supabase@1.2.3"

Updating Oden Forge

If installed via git clone

bash 
# Navigate to Oden Forge directory
cd path-to-oden-forge

# Pull latest changes
git pull origin main

# Re-run installer to update commands and scripts
./install.sh
ℹ️
The installer will overwrite existing commands and scripts but will not touch your project files, .mcp.json, or docs/ directory.

Checking for Updates

bash 
# Check if updates are available
cd path-to-oden-forge
git fetch origin
git log HEAD..origin/main --oneline

# If output is empty, you're up to date
# If output shows commits, updates are available
