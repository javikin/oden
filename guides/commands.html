Command Reference

Complete guide to all Oden Forge v3 commands
New to Oden? Start with /oden:init to set up your project, then follow the workflow: /oden:architect → /oden:spec → /oden:prd → /oden:epic → /oden:work

Setup Commands

Initialize and configure your Oden Forge environment.

/oden:init

Purpose: Interactive wizard to set up a new project with quality gates

Usage 
/oden:init

What it does:

  • Detects your tech stack (React, Node.js, Python, etc.)
  • Installs recommended MCPs for your stack
  • Sets up quality gates (80% coverage, ADR detection, etc.)
  • Creates initial documentation structure

/oden:init-mcp

Purpose: Install MCPs by category

Usage 
/oden:init-mcp essential
/oden:init-mcp design
/oden:init-mcp backend

Categories: essential, design, backend, testing, mobile, devops, productivity

/oden:help

Purpose: Show help and available commands

Usage 
/oden:help

Pre-Development Workflow

Execute these commands in order before writing any code.

/oden:architect

Purpose: Create technical decisions document and database schema

Usage 
/oden:architect

Creates:

  • docs/reference/technical-decisions.md (2000+ lines)
  • Complete database schema
  • TypeScript interfaces
  • Technology stack justification

/oden:spec [module]

Purpose: Create detailed specification for a module

Usage 
/oden:spec auth
/oden:spec payments
/oden:spec dashboard

Creates: 800+ line specification with state machines, validations, and edge cases

/oden:checklist

Purpose: Verify everything is ready before coding

Usage 
/oden:checklist

Checks: Technical decisions completeness, specs quality, documentation structure

Feature Pipeline

Turn ideas into shippable features with structured planning.

/oden:prd [feature-name]

Purpose: Create Product Requirements Document with AI brainstorming

Usage 
/oden:prd user-authentication
/oden:prd real-time-chat

Output: Comprehensive PRD with user stories, acceptance criteria, and competitive analysis

/oden:epic [feature-name]

Purpose: Convert PRD into technical epic with work streams

Usage 
/oden:epic user-authentication

Creates: Technical breakdown with parallel work streams and dependencies

/oden:tasks [epic-name]

Purpose: Decompose epic into actionable GitHub issues

Usage 
/oden:tasks user-authentication

Output: Individual tasks ready for implementation

/oden:sync [epic-name]

Purpose: Synchronize tasks with GitHub Issues

Usage 
/oden:sync user-authentication
/oden:sync status

Features: Bidirectional sync, status tracking, automatic updates

Development Commands

Quality-first development with intelligent orchestration.

/oden:work [epic/issue]

Purpose: Intelligent orchestrator for implementing features

Usage 
/oden:work user-authentication
/oden:work #123

Modes:

  • auto: Fully automated implementation
  • config: Guided implementation with checkpoints
  • smart: AI-assisted pair programming

/oden:test [subcommand]

Purpose: Testing operations with quality gates

Usage 
/oden:test gates
/oden:test coverage
/oden:test run

Quality gates: Enforces 80% coverage minimum, blocks commits without tests

/oden:debug

Purpose: Queue-based debug orchestration with persistent context

Usage 
/oden:debug

Features: Continuous session, sub-agents, maximum context preservation

Utility Commands

/oden:research [topic]

Purpose: Technical research with competitive analysis

Usage 
/oden:research "real-time sync patterns"
/oden:research "react state management 2024"

/oden:daily

Purpose: Daily progress logging

Usage 
/oden:daily

Tracks: Completed work, blockers, next priorities

/oden:review [scope]

Usage

/oden:git [subcommand]

Purpose: Git workflow management

Usage 
/oden:git status
/oden:git commit
/oden:git pr
